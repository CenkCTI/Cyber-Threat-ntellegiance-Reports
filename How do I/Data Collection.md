## Data Collection

Data collection is not random. If you just collect everything and think later, it's wrong.

Before collecting:

- Why am I collecting this?
- What exactly do I need?

If you don’t know this → you get too much useless data (information overload)

---

## Basic Principles

### 1. Define the goal

What is the question?

Example (CTI):
- Is this campaign real?
- Is this actor targeting us?

Data should serve this goal.

---

### 2. Select only relevant data

Not all data is useful.

Wrong approach:
→ “maybe useful later” → collect everything

Correct approach:
→ only collect data related to hypothesis

Because:
- Data has cost (time, effort)
- Too much data = harder analysis

---

### 3. Choose collection method

Different ways:

- Logs (SIEM, EDR)
- OSINT
- Reports
- Observation
- Interviews (HUMINT)

Need to decide:
- who collects
- where
- when
- how

---

### 4. Plan analysis early

Before collecting:

- how will I analyze?
- what tools / methods?

Good analysts think analysis before data collection.

---

### 5. Store and use data properly

- Who will use it?
- Where is it stored?
- How to access?

In CTI → sharing matters (SOC teams, reports)

---

## Data Quality (Very Important)

Good data should be:

- **Accurate** → not biased  
- **Reliable** → same result if repeated  
- **Usable** → others can verify  
- **Useful** → helps solve problem  
- **Complete** → not missing key parts  

Bad data → bad intelligence

---

## What Can Go Wrong

### Source problems
- Logs incomplete  
- People lie (interviews)  
- Observation bias  

### Collector problems
- Not trained  
- Biased  
- Decides what is “important” → can miss things  

### Method problems
- Bad questions  
- Bad forms  
- Confusing structure  

---

## Key Idea (Important)

More data ≠ better intelligence

Sometimes:
→ more data = more confusion

Analyst must:
- filter
- question
- validate

---

## Data Collection Plan (Important Concept)

Before starting:

1. What do I need?
2. What do I already know?
3. What is missing?

Then create a simple plan.

Good plan:
- saves time  
- prevents useless data  
- helps analysis  

---

## Techniques (Short)

Common methods:

- Written sources (reports, articles)  
- Observation (watch behavior)  
- Content analysis (analyze text, keywords)  
- Surveys  
- Interviews  

In CTI:
→ mostly logs + OSINT + reports + some HUMINT

---

## Final Thought

Collectors don’t collect intelligence.

They collect:
- data
- information

Intelligence comes later → during analysis

---

CTI mindset:

- Don’t collect everything  
- Don’t trust everything  
- Always question data
